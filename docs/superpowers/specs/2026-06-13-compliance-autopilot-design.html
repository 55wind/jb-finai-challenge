1배경 — 은행 광고심의의 "수정 왕복" 병목

은행의 대고객 광고·안내문은 게시 전 내부 준법감시인의 사전심의가 의무다(금융소비자보호법 §22, 은행연합회 광고심의규정). 현실의 흐름은 다음과 같다.

마케팅팀 초안 → 준법심의(지적) → 마케팅팀 수정 → 재심의(또 지적) → 재수정 → … → 승인 → 게시
                └──────────────── 이 왕복이 수일~수주, 캠페인 적기를 놓치는 핵심 병목 ────────────────┘

기존 JB 준법 코파일럿은 이 중 한 번의 심의를 자동화했다. 그러나 "수정→재심의" 왕복 자체는 여전히 사람 몫이고, 자동 치환안이 없는 비정형 위반(예: 근거 없는 최상급 "최고 금리")은 수동으로도 통과까지 못 간다(데모상 75·주의에서 정체). 오토파일럿은 이 왕복 전체를 에이전트가 자율 수행하여 병목을 제거한다.

3은행(JB) 맥락 반영 핵심

본 기능은 일반 카피 교정기가 아니라 지방은행 지주의 준법 거버넌스에 맞춰 설계한다.

3.1 지방은행 특성 — 다국어·계열 확장

  • JB금융그룹은 전북은행·광주은행 등을 둔 지방은행 지주다. 특히 전북은행은 외국인 고객(베트남·중국 등) 비중이 높아, 오토파일럿은 다국어 초안도 동일 루프로 개선(EN·VI·ZH·JA 룰셋 재사용)한다 — 실제 사업 연계성.
  • 확장 경로: 동일 엔진을 광주은행·JB우리캐피탈 등 계열, 그리고 채널(앱 푸시·문자·지점 게시물)로 수평 확장.

3.2 책임소재 — 자율 수정 ≠ 자율 게시

은행 원칙: 최종 책임은 사람오토파일럿은 통과 가능한 초안을 제안할 뿐, 결코 자동 게시하지 않는다. 결과물은 pending 상태로 준법관리자 승인 큐에 올라가며, 사람의 승인(approve)이 있어야 "배포가능"이 된다. AI가 무엇을 고쳤는지와 사람이 무엇을 승인했는지가 분리·기록된다.

3.3 내부통제 — 감사추적 의무

은행은 금융감독원 검사 대응을 위해 "누가/언제/어떤 근거로" 광고를 변경·승인했는지 재구성할 수 있어야 한다. 오토파일럿의 모든 자동 수정 회차를 감사로그에 적재한다(§8).

3.4 환각 리스크 = 은행에 치명적

AI가 잘못된 고지를 삽입하거나 위반을 놓치면 불완전판매·과징금으로 직결된다. 따라서 루프의 합격 판정은 비결정적 LLM이 아니라 결정적 룰엔진이 내린다(§4). 에이전트는 스스로 "통과했다"고 선언할 수 없다.

4에이전트 루프 아키텍처

입력: draft₀ (마케팅 초안), content_type, language, product_facts(상품 실제 조건)

for i in range(MAX_ITER = 4):
    reportᵢ = run_review(draftᵢ)                 # ① 판단  (기존 파이프라인 재사용)
    if reportᵢ.grade == "pass":  return ✅ 수렴   # ④ 종료조건 (목표 달성)

    # ② 행동 — 2단계 수정 (결정적 우선 → LLM 보강)
    draft' = apply_rule_fixes(draftᵢ, reportᵢ)    #   (a) 룰 자동치환·고지추가 (무료·결정적)
    residual = re_review(draft').위반             #       남은 비정형 위반만 추출
    draftᵢ₊₁ = remediation_agent(draft', residual, retrieval)  # (b) LLM 재작성 (필요시에만)

    # ③ 검증/개선 — 진전 없으면 중단 (무한루프·발산 방지)
    if draftᵢ₊₁ == draftᵢ:           return ⏹ 수렴실패(텍스트 불변)
    if scoreᵢ₊₁ ≤ scoreᵢ:           return ⏹ 수렴실패(점수 정체)

return ⏹ 미수렴 → 사람 에스컬레이션 (도달점수 + 잔여위반 N건 핸드오프)

출력: trace = [{iter, draft, report, changes[{quote, fix, basis_id, engine}]}], converged, final_grade

하이브리드 수정(룰 먼저 → LLM은 잔여만)의 근거: 정형 위반은 결정적 치환이 빠르고 무료이며 재현 가능하다. LLM은 자동 치환안이 없는 비정형 위반(최상급·맥락 오인 등)에만 호출되어 비용·지연·환각 표면적을 최소화한다.

5수정 에이전트 (remediation_agent) — 멀티에이전트 맛

잔여 위반을 고쳐쓰는 주체는 단순 치환기가 아니라 "마케팅 효과 유지" 제약을 받는 에이전트다. 마케터 관점(상품 매력 보존)과 준법 관점(위반 제거)을 한 프롬프트에 긴장 관계로 넣는다.

system: "당신은 은행 마케팅 카피라이터이자 준법 보조자다.
         아래 [위반]만 규제근거에 맞게 고쳐써라. 제약:
         · 상품의 핵심 셀링포인트(금리 수치·상품명)는 보존
         · 새 위반·허위 고지 생성 금지
         · 필수고지는 본문에 없을 때만 추가
         JSON: {revised_text, changes:[{before, after, basis_id, rationale}]}"
user:   [현재 초안] + [잔여 위반(문구·근거·제안)] + [규제 근거(RAG)]
  • 인용 제한: basis_id는 검색된 조항 화이트리스트로 한정(기존 가드 재사용) → 조항 창작 차단.
  • 검증은 룰엔진: 에이전트 출력은 다음 회차 run_review로 재검증된다. 에이전트가 "고쳤다"고 주장해도 룰엔진이 통과를 인정해야만 수렴.
  • 폴백: LLM 불가 시 (b)단계를 건너뛰고 (a) 룰 치환만으로 루프 → 점수는 오르되 비정형 위반은 잔존하여 "사람 에스컬레이션"으로 종료.

11테스트 (TDD)

  • 수렴: FX-01(danger) 오토파일럿 → 점수 단조 증가, 등급이 danger 밖으로 개선, MAX_ITER 이내.
  • 종료 보장: 어떤 입력도 MAX_ITER를 넘지 않음 · 정체/불변 시 즉시 종료.
  • 합격 무결성: 수렴(converged=true)은 룰엔진 grade==pass일 때만.
  • 폴백 모드: LLM 폴백(결정적)에서도 점수가 개선되고 루프가 종료(미수렴은 에스컬레이션).
  • 휴먼인더루프: 오토파일럿 결과는 자동 승인되지 않음(상태 pending 유지).
