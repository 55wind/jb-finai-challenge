1개요 · 설계 원칙

백엔드는 프론트와 AI 모델 사이에서 명시적이고 설명가능한 심의 파이프라인을 오케스트레이션한다. LangChain 같은 블랙박스 에이전트 프레임워크 대신 일반 파이썬 함수의 명시적 호출 그래프로 구성해 각 단계를 추적·테스트·감사할 수 있게 했고, LLM 미연결 시 결정적 룰엔진으로 폴백해 오프라인에서도 전 기능이 동작한다. 모든 제출·승인·배포는 SQLite 감사로그에 영구 기록된다.

2오케스트레이션 파이프라인

orchestrator.run_review()가 단일 심의 요청을 처리하는 핵심 흐름:

run_review(text, content_type, language, product_facts, quick):
  0. 길이 가드   len(text) < MIN_TEXT_LEN(2) → skip ("너무 짧음")
  ① 유형 분류    classify(text)  — 수동 지정 시 분류기는 참고로만
  내규 주입      _internal_rules_loader()  — 활성 내규를 법규와 함께 적용
  ②∥③ 병렬      asyncio.gather(
                   retrieve(text, ctype),              # RAG
                   run_rules(text, ctype, lang, intl)  # 룰엔진(법규+내규)
                 )
  ④∥⑤ 분기      if not quick and LLM 가용:
                   asyncio.gather(llm_reviewer.review(...), simulator.simulate(...))
                 else:  # quick 모드 또는 LLM 미연결
                   결정적 폴백(룰 기반 심의의견 + 결정적 시뮬레이션)
  ⑥ 종합        report.compose(...) → 점수·등급·하이라이트·인용
  다국어 방어    language ∈ {vi,zh,ja} → language_confidence="low" + 경고문
설정값값근거
MIN_TEXT_LEN2"원금 보장", "확정 연 5%" 같은 짧은 위험 구절도 즉시 검사. 2자 미만만 스킵
quick 모드bool실시간 타이핑용 — LLM(④⑤) 생략하고 룰+RAG+결정적 시뮬만 → 밀림 방지(프론트 0.7초 디바운스와 연동)
LOW_CONFIDENCE_LANGS{vi,zh,ja}로컬 7B가 한/영 대비 뉘앙스 신뢰도 낮은 저자원 언어 → LLM 출력은 참고로만, 룰엔진을 권위로 + 사람검토 권장
왜 명시적 파이프라인인가평가 기준의 "AI Agent 정의(판단·행동·검증·종료조건)"를 프레임워크 마법이 아니라 읽히는 코드로 구현했다. 각 단계가 일반 함수라 단위 테스트·감사·근거 추적이 가능하고, 금감원 검사 시 "이 등급이 나온 경로"를 그대로 제시할 수 있다.

4.1 책임소재 분리 기록

제출 시 AI 자동개선과 사람의 제출을 별도 감사 엔트리로 분리한다 — 내부통제·금감원 검사 대응:

create_submission():
  if autopilot_summary:
    log(sid, actor="AI-오토파일럿", action="auto_remediate", detail=요약)   # AI가 한 일
  log(sid, actor=submitter, action="submit", detail="AI 1차심의 🟢통과(88점) — 검토 요청")  # 사람이 한 일
decide(approve): log(actor="준법관리자", action="approve", "승인 — 배포가능 상태 전환")
distribute():    log(actor="배포시스템", action="distribute", "마케팅 채널 자동 배포: ...")
분리 기록의 의미감사로그에 "AI 자동개선 → 사람 승인 → 시스템 배포"가 각각 다른 actor로 남아, "AI가 자동으로 게시했다"가 아니라 "사람이 승인해 배포됐다"는 책임 경계가 데이터로 증명된다.

5망분리(Air-gap) 아키텍처 — 3-zone

┌─ 내부망 (Air-gap) ────────────┐   ┌─ DMZ 망연계 게이트웨이 ───┐   ┌─ 외부망 ───────────┐
│ 심의 코어: LLM·RAG·룰엔진·     │   │ 단방향/배치 전송·검역       │   │ 법제처 OpenAPI(F14) │
│ 시뮬레이터·승인·감사로그       │←──│  · 규제 피드 수신(배치)     │←──│ 고시 RSS            │
│ (심의 데이터 외부 유출 0)      │──→│  · 승인본 발송(일방향)      │──→│ FCM·SNS·알림톡(F15) │
└───────────────────────────────┘   └────────────────────────────┘   │ 코어뱅킹 상품DB(F16) │
                                                                       └────────────────────┘
  • 심의 코어(모델·룰·심의 데이터)는 내부망 격리 — 미공개 마케팅 자료·내규가 외부로 나가지 않음(외부 API 0).
  • 외부 연동(F14·F15·F16)은 코어가 직접 통신하지 않고 어댑터 인터페이스 → DMZ 게이트웨이 경유. 데모는 목/피드, 운영은 실 API 교체.
  • 규제 피드는 배치 수신(실시간 크롤 아님), 승인본은 일방향 발송 — 검역·단방향 전송으로 데이터 유출 차단.

6폴백 전략 — 오프라인 전체 동작

단계LLM 모드폴백(LLM 미연결)
② RAGBGE-m3 임베딩(설치 시)어휘 검색(토큰 중첩+카테고리 부스팅, 의존성 0)
③ 룰엔진항상 결정적 — LLM 무관
④ LLM 심의Qwen2.5 뉘앙스 심의룰엔진 결과를 심의의견 형태로 결정적 변환
⑤ 시뮬레이터페르소나 5종 1인칭 역할극(+전북 특화)룰 카테고리→페르소나 취약점 매핑 템플릿(5종)
오토파일럿LLM 재작성결정적 룰 치환 + 고위험 표현 제거(안전망)까지 동작, 잔여는 사람 에스컬레이션

근거: 대회 평가환경에 GPU/Ollama가 없을 수 있다. 폴백 모드는 "축소판"이 아니라 전체 데모 플로우(심의→시뮬→수정→승인→배포→다국어→감사)를 결정적으로 재현한다. 동시에 테스트가 결정적이 되어 회귀 검증이 가능하다(69개 테스트는 폴백 고정).

8비기능 요구 · 견고성

  • 병렬성: asyncio.gather + 동기 함수는 asyncio.to_thread로 오프로드(RAG·룰엔진) — 이벤트 루프 블로킹 방지.
  • 입력 검증: Pydantic 모델로 타입·범위·enum 강제. 미존재 리소스 404, 상태 위반 409(미승인 배포/번역).
  • 멱등·결정성: 폴백 경로는 동일 입력→동일 출력 → 테스트·시연 재현성.
  • 관측성: /api/health로 엔진 모드 노출, 감사로그로 전 액션 추적.
  • 보안: 고객 개인정보 미취급(콘텐츠 텍스트만), 외부 전송 0, 감사 불변 기록.
