1에이전트 개요 · 정의 부합

본 시스템의 AI는 단순 분류기가 아니라 목표(준법 통과)를 향해 판단·행동·검증·개선을 자율 반복하는 에이전트다. 핵심 안전장치는 판정 권위를 결정적 룰엔진에 두는 것 — LLM은 뉘앙스 심의·재작성·역할극을 담당하되, 합격 여부는 항상 룰엔진(YAML·정규식)이 심판한다. 따라서 LLM이 "통과했다"고 환각해도 등급을 위장할 수 없다.

2① 콘텐츠 유형 분류기 (F1)

결정적 예금·투자·대출을 다국어 키워드 가중치로 분류. classify() → {content_type, confidence, scores}.

설정값근거
키워드 가중치핵심어 2점 / 보조어 1점"예금·펀드·대출" 등 직접 신호는 2점, "세전·승인" 등 정황 신호는 1점으로 차등 → 오분류 억제
다국어 패턴KO·EN·VI·ZH·JA 정규식전북 외국인 고객 콘텐츠도 유형 라우팅 — 언어별 룰셋 진입점
무신호 기본값deposit총점 0이면 가장 보편적·보수적 유형으로. 수동 지정이 항상 우선(분류기는 참고)

3② 규제 근거 검색 / RAG (F2)

결정적 기본 어휘 검색(토큰 중첩 + 카테고리 키워드 부스팅), sentence-transformers 설치 시 BGE-m3 임베딩 코사인으로 자동 전환. 인용은 검색된 코퍼스 조항으로만 제한.

설정값값근거
SCORE_THRESHOLD0.05이 미만은 "근거 불충분" 처리 → 약한 매치로 억지 인용하는 환각 방지. 짧은 광고 문구도 관련 조항을 잡되 무관 조항은 배제하는 균형점
top_k4상위 4개 조항이 한 화면 인용에 적정 — 너무 많으면 LLM 프롬프트·UI 과부하, 적으면 교차참조 누락
참조 그래프 확장연계 조항 0.5배 가중법률은 의미 유사성이 아니라 참조 트리(조→시행령→고시)로 얽힘. 검색 조항이 참조하는 규정을 맥락으로 추가하되, 직접 매치보다 낮은 점수로
질의 확장카테고리 시드 키워드"확정 5%" 같은 짧은 입력에 "보장·확실·오인" 등 시드를 더해 어휘 검색 재현율 보강
임베딩 모델BAAI/bge-m3 (선택)MIT·다국어 강함. 미설치 시 어휘검색 폴백 → 의존성 0으로 즉시 구동
등급 권위는 RAG가 아니다RAG 오검색이 나도 준법 등급은 결정적 룰엔진이 정하므로, 잘못된 검색은 LLM '참고 의견'에만 영향을 줄 뿐 등급을 흔들지 못한다 — RAG 실패의 폭발 반경을 구조적으로 제한.

4③ 룰엔진 — 결정적 심의 백본 (F3)

결정적 YAML 룰(KO 8종 + EN 재심의 2종 + 내규) + 정규식. 준법 등급의 판정 권위.

  • forbidden 룰: 패턴 매치 시 위반 + span 추출(에디터 밑줄). condition_absent_any로 면제 조건(이미 고지 있으면 제외) 지원.
  • required 룰: requires_any 패턴이 전부 부재하면 위반 → fix_append로 고지문 추가 제안.
  • 수정안 생성: "확정 연 5%"의 숫자를 보존한 치환안(예: "연 5%(세전, 변동가능)") 등 룰별 suggestion_map.

코드 수정 없이 rules.yaml에 룰 추가만으로 확장 가능 — 결정적·설명가능·테스트 가능(TDD 정답셋).

5④ LLM 뉘앙스 심의 (F4)

LLM RAG 근거 + 룰엔진 결과를 주입받아 룰이 놓친 뉘앙스 리스크(맥락상 오인, 조건 축약, 과장 분위기)를 도출. 한국 준법감시인 페르소나, JSON 강제 출력.

환각 방어 장치구현
인용 화이트리스트basis_id가 검색된 조항 id 집합에 없으면 제거(null) — 존재하지 않는 조항 창작 차단
quote 원문 검증지적 문구가 초안 원문에 실제로 없으면 비표시 — "없는 문구 지적" 차단
과탐 억제 프롬프트"준수하면 빈 배열, 억지로 만들지 말 것 · 이미 포함된 고지를 누락으로 지적 말 것" 명시
점수 영향 상한LLM 단독 지적은 _LLM_CAP=6점까지만 감점 + advisory(참고의견)로 분리 — 룰엔진을 권위로
폴백JSON 파싱 실패·LLM 불가 시 룰엔진 결과를 심의의견으로 결정적 변환

6⑤ ★소비자 오인 시뮬레이터 (F5) — 헤드라인

LLM 페르소나 5종(고령자·금융초보·외국인 + 전북 특화 농업인·소상공인)이 광고를 1인칭으로 읽고, 자신이 믿게 된 것 중 ⓐ상품 실제조건 또는 ⓑ규제상 암시 금지 항목과 어긋나는 '오해'를 스스로 식별한다. 오해 판정기가 위험 오해를 플래그하고 유발 문구를 역추적.

설계 결정구현근거
3 페르소나 1콜 처리역할극+판정을 단일 LLM 호출로로컬 7B 큐 적체·레이턴시 방지 — 3콜이면 작성 흐름이 밀림
프로필 이해도 제약"금융이해도를 벗어난 전문 해석 금지"72세 은퇴자가 전문가처럼 읽으면 시뮬 의미 상실 — 취약성 재현이 목적
오해만 수집실제조건과 일치하는 올바른 이해는 제외, 없으면 빈 배열과잉 경고 억제 — "진짜 오해"만 헤드라인에
유발문구 원문 검증trigger_phrase가 본문에 없으면 null역추적 클릭 이동의 정확성 + 환각 차단

폴백: LLM 불가 시 룰 카테고리 → 페르소나 취약점(vulnerabilities) 매핑 템플릿으로 결정적 시뮬레이션 — 오프라인에서도 헤드라인 재현.

7⑥ 리포트 종합 — 점수·등급 (F6)

룰·LLM·시뮬 결과를 통합. 감점식 점수(100 시작) + 등급 판정. 핵심은 "룰엔진이 권위, LLM·시뮬은 보정만".

감점 계수high / medium / low상한
룰 위반 _RULE_PENALTY12 / 4 / 1무제한 (권위)
LLM 단독 _LLM_PENALTY5 / 2 / 1_LLM_CAP=6
시뮬 _SIM_PENALTY4 / 2 / —_SIM_CAP=12

근거: 룰 high(12점)는 LLM high(5점)·시뮬 high(4점)보다 훨씬 무겁다 — 결정적 위반이 추정성 지적을 압도하게. LLM·시뮬은 상한(6·12)을 둬 "참고 의견이 등급을 뒤집지 못하게" 한다. 점수 하한 5점.

7.1 등급 판정 로직

if 룰 위반 0건 and rule_penalty==0:        # 결정적으로 준법
    grade = pass(≥80) | caution            # 페르소나·LLM만으론 위험 강등 불가
elif score<60 or (high≥1 and score<75): grade = danger 🔴
elif score≥85 and high==0:                grade = pass 🟢
else:                                     grade = caution 🟡
설계 핵심"룰 위반이 없으면 LLM·시뮬레이터 참고의견만으로 위험 등급을 줄 수 없다"는 규칙이 과잉규제와 환각 등급 조작을 동시에 차단한다. 합법이지만 오해 유발인 콘텐츠는 반려가 아니라 "표현 개선 권고".

8★ 준법 오토파일럿 — 자율 개선 루프 (F12)

LLM+결정적 위반 초안을 심의→고쳐쓰기→재심의로 통과/종료조건까지 자율 반복하는 목표지향 에이전트. 합격 판정은 결정적 룰엔진(run_review의 grade)만 — LLM이 합격을 위장할 수 없다.

8.1 한 회차의 행동 순서

for i in range(max_iter):
  report = run_review(current)              # 재심의(판단)
  if score < prev_score: stop "regressed"   # 발산 방지 — 직전 결과 유지
  if grade == "pass": converged; stop "passed"
  if score == prev_score: stop "plateau"    # 정체 — 진전 없음
  ── 행동 ──
  (a) det_fixes  : 결정적 룰 치환·고지추가 적용
  (b) _strip_residual_high : LLM이 못 없앤 고위험 금지문구 안전망 제거(위험 등급 탈출 보장)
  (c) remediate  : 잔여(비정형) 위반만 LLM 재작성(셀링포인트 보존)
  (d) _dedup_disclosures : 중복 고지문 정리
  if nxt == current: stop "no_change"       # 불변 — 더 개선 불가
설정 / 종료조건값근거
AUTOPILOT_MAX_ITER4 (요청 1~6)로컬 7B 4회면 대개 수렴 + 무한루프·과도 레이턴시 방지. 종료보장의 상한
passedgrade==pass목표 달성 — 결정적 룰엔진이 통과 선언
regressedscore↓발산 시 이 회차 버리고 직전(더 나은) 결과 유지 — 개악 방지
plateau / no_change점수·텍스트 불변진전 없으면 즉시 종료 → 사람 에스컬레이션
합격 무결성grade는 run_review만LLM 재작성이 "통과"라 주장해도 룰엔진 재심의로만 합격 — 환각 위장 불가

검증 결과(데모): 🔴46 → 🟡82 → 🟢88로 자율 수렴. 최종 게시는 pending으로만 제출되어 사람 승인 필수, AI 자동개선 이력은 감사로그에 분리 기록.

8.2 수정 에이전트 (remediator) 제약

  • 셀링포인트 보존: "금리 수치·상품명·혜택은 최대한 보존" — 과교정으로 마케팅 가치 훼손 방지.
  • 중복 고지 금지: 필수 고지문은 룰엔진이 처리하므로 LLM은 새로 추가 안 함(중복 방지).
  • 인용 제한: basis_id는 검색 조항으로만.
  • 폴백: LLM 불가 시 변경 없이 반환 → 루프가 "수렴 실패 → 사람 에스컬레이션"으로 안전 종료.

8.3 결과 표시 — 수정 내역·가독성 (결정적, 피드백 반영)

  • 수정 전/후 비교 뷰(diff) 결정적: 초안→최종을 어절 단위로 비교(textdiff.word_diff, difflib 기반)해 삭제·추가 세그먼트로 표시 — 준법감시인·마케터가 AI가 무엇을 바꿨는지 한눈에 파악, 감사 추적 강화. LLM 불필요.
  • 가독성 참고지표 결정적: 자동수정 완료 후 1회 표시(readability.score — 문장 길이·전문용어 밀도). 준법 점수와 별개의 참고지표이며 마케팅 효과를 예측하지 않는다 — 준법 통과만 추구할 때의 문장 경직을 작성자가 인지하도록 보조하는 신호로 한정.

9다국어 생성 + 재심의 (F10)

승인본(KO) → 타깃언어 생성 후 동일 파이프라인으로 재심의. 직역이 만드는 보장성 오역(예: "확정 5%"→"Guaranteed 5%")을 영문 룰셋이 검출. 한국어가 주력 시장이므로 우선 배포하고 다국어는 후속(생성·심의 한계비용 ≈ 0)이라 영어 재심의가 한국어 발송을 지연시키지 않는다. 재심의 불통과 시 needs_attention 플래그 → 마케터·준법감시인 알림(감사로그) + 외국어 채널 배포 보류. 비-한/영 언어는 language_confidence=low 플래그 + 사람검토 권장(KO 피벗 계획).

10내규 자연어→룰 (F13) · 규제 추적 (F14)
