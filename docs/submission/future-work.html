향후 발전 방향 (Future Work)

JB 준법 코파일럿 — 개선 계획 · 모델 고도화 · 운영 적용 고려사항

예선 MVP는 F1–F16과 자동 테스트 57건으로 핵심 흐름(작성→오인 시뮬레이션→자동 개선→승인→배포→다국어)을 동작으로 증명했다. 본선·운영 단계의 발전 방향을 세 축으로 정리한다.

1개선 예정 기능 및 추가 데이터 확보 계획

정답셋·실데이터를 확보해 정확도와 커버리지를 끌어올린다.
  • 룰 커버리지·거버넌스 — 룰별 양성/음성 정답셋(픽스처) 확장과 커버리지 리포트, 미탐 사례(LLM·사람이 잡은 것)를 신규 룰로 제안하는 피드백 루프 구축본선
  • 형태소 기반 매칭 — 띄어쓰기·이형태·동의어 변형까지 잡도록 정규식을 형태소 분석으로 보강(룰 미탐 축소)본선
  • 실 심의 정답셋 확보 — 준법감시부 과거 심의 사례 50–100건을 블라인드 비교(AI vs 사람)해 정밀도·재현율 측정 및 점수 가중치 캘리브레이션PoC
  • 규제·상품 실데이터 연동 — 규제 피드를 법제처 국가법령정보 OpenAPI·금융위 고시로, 상품조건을 코어뱅킹 API로 교체(현재 목/피드 어댑터)PoC
  • 내규 룰 검증 강화 — 적용 전 룰 테스트·미리보기(F13)에 과탐/미탐 통계와 충돌 룰 탐지 추가

2모델 고도화 및 서비스 확장 아이디어

온프레미스 제약을 지키며 모델·적용 범위를 넓힌다.
  • 모델 업그레이드·파인튜닝 — Qwen2.5-7B → 14B-Q4(환경변수 1개 교체) 또는 준법 심의 데이터 LoRA 파인튜닝으로 도메인 정확도 향상본선
  • 임베딩 기본 탑재 — BGE-m3 임베딩 RAG를 기본화하고 리랭커를 더해 근거 검색 정밀도 강화(현재 어휘검색 폴백)
  • 다국어 정식 확장 — 영문에 더해 베트남어·중국어·일본어 룰셋·고지문을 정식화(전북 외국인 거주자 고객 대응)
  • 적용 범위 확장 — 광고·안내문을 넘어 약관·상품설명서·민원 응대 초안·내부통제 점검까지 동일 파이프라인으로 확대
  • 채널 확장 — 배포 Last-Mile(F15)을 영업점 단말·홈페이지 배너·앱 내 고지 등으로 넓혀 전사 일관 심의

3실제 운영 환경 적용 시 고려사항

금융 망분리·책임·성능 요건을 충족해야 실서비스로 전환된다.
  • 망분리·인프라 사이징 — 온프레미스 GPU 서버에 OLLAMA_NUM_PARALLEL·KEEP_ALIVE·VRAM(8–12GB)를 동시성(심의당 2콜) 기준으로 산정, 미공개 마케팅 자료의 외부 전송 0 보장
  • 책임 한계·휴먼인더루프 — AI는 1차심의 보조이며 최종 판단·법적 책임은 준법감시인. 승인 전 원문 직접 검토 확인과 감사로그로 책임 소재를 명확화
  • 룰·모델 거버넌스 — 룰 변경은 사람 승인 게이트·버전관리·감사를 거치고, 모델 교체 시 회귀 정답셋으로 영향 검증
  • 성능·가용성 — LLM 지연·장애 시 룰 기반 결정적 폴백으로 무중단 동작, 응답 SLA와 정밀도/재현율을 상시 모니터링
  • 단계적 도입 — PoC(블라인드 비교) → 1개 부서 파일럿 → 전사 순으로 적용하며 검출률·시간 절감을 정량 측정
